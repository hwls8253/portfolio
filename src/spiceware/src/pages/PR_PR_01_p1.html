PRODUCT

소개

클라우드 환경에 최적화된 솔루션 서비스를 제공하는 Spiceware
  • 생산성
    • Filter 방식 기반으로 API방식 대비 생산성 향상
    • DB의 종류에 상관없이 기존 성능 유지하여 암호화 적용
    • 평문과 암호문 선택적 선별(선택적 데이터 마이그레이션)
    • 코드형 인프라 도입으로 1일 이내 사용가능 (퍼블릭)
    • TIME & COST SAVING
  • 안전성
    • 최소 128bit 이상의 검증된 알고리즘
    • 시스템 별 개별알고리즘 적용
    • Row별 다중 암호 키(key) 방식
    • 암호화와 비식별화 동시 지원
    • 전송구간 내 데이터 암호화
    • 형태 보존 암호화 방식 지원
  • 성능과 편의성
    • Auto Scale-out (퍼블릭)
    • Docker Container 지원
    • 통합모니터링 기능 제공 (Restful API 제공)
    • 어플리케이션 레벨에서 데이터 암호화 관리
    • 기존 API방식 이상의 빠른 성능

ISMS지원 사항

암호화 대상

  • HTTP Filter Mode

    완전 관리형 Database 전 제품, 일반 RDBMS 제품, Big Data용 제품, NoSQL 및 Storage 등 제한없이 모두지원

  • SQL Filter Mode / JDBC 지원 DB

    Microsoft SQL Server, MySQL, MariaDB, PostgreSQL, Amazon Aurora, Amazon RDS, Oracle

암호화 방식

  • Spice.agent (javaagent)
  • Gateway (EC2 / Auto Scale Out, 19’3 Q)
  • API 방식 (API Access Code 인증, 19’3 Q)

알고리즘

  • 양방향 : AES-256, ARIA-2526, SEED-128, FPE (NIST Special Publication 800-38G)
  • 단방향 : SHA-256 / 384 / 512
