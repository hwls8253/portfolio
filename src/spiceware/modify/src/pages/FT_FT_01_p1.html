FEATURE

특징

  • 새로운 필터방식
  • 점진적인 암호화
  • 전송구간 암호화
  • 암호화엔진 분리
  • Auto Scale-Out
  • 통합 모니터링
  • 무차별 대입공격
  • 암호키 자동업데이트
  • 비식별화
  • 형태보존 암호화

  • Filter 방식 기반으로 API방식 대비 생산성 향상

    어플리케이션(웹 서비스) 의 앞과 뒤에서 input parameter와 output SQL Query에서 주요 데이터를 Filter하여
    어플리케이션 수정 없이 암 복호화가 가능하여，수작업으로 인한 보안 취약점이 발생하지 않도록 전 공정이 자동화 되었습니다.

  • 점진적인 암호화 (무중단 데이터 마이그레이션)

    무중단 또는 긴급 적용 시스템인 경우에도, 대상 DB에 평문과 암호가 섞여 있는 경우에도
    암호 자동 식별가능 데이터 마이그레이션은 배치로 적용 할 수 있습니다.

  • 전송 구간 암호화

    접근 제어된 네트워크로 전송되는 암호키는 전송구간 암호화 (SSL)과 전문 자체 암호화로 이중 보호 됩니다.

  • 암호화 엔진 분리

    고객의 시스템 상황에 맞게 논리적으로 구성한 시스템 별로 개별 알고리즘과 개별 암호키를 적용이 가능합니다.
    데이터 암호화의 확산(Diffusion)과 혼돈(Confusion) 을 간단히 실현하여, 더욱 더 안전하게 데이터를 보호합니다.

  • 부하 분산 지원

    Spiceware on AWS의 Agent는 Auto Scale Out 이 발생하여도 추가 연결 작업 없이 자동으로 기능을 수행하고,
    수행 결과는 자동으로 통합 수집 모니터링이 가능합니다.

  • 통합 모니터링

    실시간 모니터링, 암호화 설정, 처리 이력 등의 기능을 통해 전사에서 수행되는 데이터 보안을
    웹을 이용하여 한눈에 파악이 가능합니다.

  • 무차별 대입 공격(bruteforceattack)방어를 위한 여러가지 장치

    • 스파이스웨어의 암호는 최소 128bit 이상의 보안 강도를 가짐
    • Table 내 같은 Column 이라도 Row 별로 개별 ship_addr 암호키를적용 가능
    • 암호문 내에 메타 정보 구간이 포함되어 있어 무차별 대입 공격(brute force attack)방식으로 복호화는 불가능합니다.

  • 자동화된 데이터 보안 (마스킹 기능 연계)

    MVC패턴으로 개발된 웹서비스 시스템에서 암호화,복호화,데이터마스킹을 코딩없이 설정만으로 한번에 제공하여 개발자의 생산성을 높임과 동시에 데이터보안을 완성 합니다.

  • 부분 암호화 처리 응용 방안

    부분 암호화 기능을 제공하여 빅데이터 활용 시 평문 변환 구간으로 인한 사각지대를 제거하여 데이터 유출 방지

  • 형태 보존 암호화

    Spiceware on AWS 에 탑재된 FPE 알고리즘은 NIST FF1 표준 (NIST Special Publication 800-38G)을 준수하고, 평문을 동일 Unicode Set별로 인지 후 암호화하여 Unicode 4.0에서 지원하는 모든 언어를 지원합니다.
    (숫자-영어, 한글-기호, 한글-영어, 한자-한글, 숫자-한글, 등)

    “데이터 사이즈 변경없이 데이터의 형태는 보존하면서 암호화하는 기술로 상담 메모에 개인정보가 섞인 경우 적합한 암호화 방법입니다.”
